System Overview

The Purchase Recording System captures user interactions on e-commerce websites and replays them automatically for purchase automation. The system records mouse clicks, keyboard inputs, and navigation actions, then substitutes real customer data during playback.

How it works:
1. Record purchase flow using dummy data
2. System captures all interactions
3. During playback, dummy values are replaced with real customer data
4. Purchase completes automatically using real billing information

Setup & Prerequisites

Required Tools
  • Node.js (v16 or higher)
  • Chrome or Chromium browser
  • Terminal/Command prompt access
Installation
cd recording_system npm install npx playwright install chromium
Starting the Recording System
node index.js --help

This will show all available commands and options.

Recording a Purchase Flow

Step-by-Step Recording Process
1
Start Recording
node index.js record https://partstown.com/product-page

The system will auto-generate filename: ./recordings/partstown.json

2
Navigate the Site

The browser will open automatically. Navigate through the purchase flow exactly as a customer would:

  • Add product to cart
  • Go to checkout
  • Fill out all forms using the dummy values above
  • Complete the entire flow but STOP before final purchase submission
3
Stop Recording

Press Ctrl+C in the terminal to stop recording. The system will save the recording file.

Important: Do NOT complete the actual purchase during recording! Stop just before the final "Place Order" or "Submit Payment" button.
What Gets Recorded
  • Mouse Clicks: All button clicks and element interactions
  • Text Input: Form fields and text entry
  • Navigation: Page transitions and URL changes
  • Scrolling: Page scroll positions
  • Window Events: Resize and focus changes

Testing Recordings

Basic Playback Test
node index.js play https://partstown.com/different-product --slow-mo 2000

This replays the recording on a different product page with 2-second delays between actions.

Test with Variables
node index.js play partstown.json --vars-file test-variables.json --headless
Creating Test Variables

Create a test-variables.json file with different data to test variable substitution:

{ "first_name": "Jane", "last_name": "Doe", "email": "jane.doe@testcompany.com", "phone": "555-987-6543", "company": "Test Company LLC", "address": "456 Oak Avenue", "city": "Chicago", "state": "Illinois", "zip_code": "60601" }
Debugging Failed Recordings
node index.js play partstown.json --slow-mo 5000 --ignore-errors

Use slow motion and error ignoring to see what's happening when recordings fail.

Troubleshooting

Common Issues
Recording fails to start:
  • Check that Playwright is installed: npx playwright install
  • Ensure the URL is accessible
  • Try with --headless=false to see the browser
Playback doesn't work:
  • Website layout may have changed since recording
  • Try using --slow-mo 2000 for slower playback
  • Check that dummy values were used correctly during recording
  • Use --ignore-errors to see where it stops
Variables not substituting:
  • Ensure exact dummy values were used during recording
  • Check variable names match between recording and substitution
  • Verify JSON syntax in variables file

Best Practices

Recording Best Practices:
  • Use consistent dummy values across all recordings
  • Record the complete flow but stop before payment
  • Test recordings immediately after creation
  • Use descriptive filenames for recordings
  • Record during off-peak hours to avoid site changes
Testing Best Practices:
  • Always test with different products/URLs
  • Test variable substitution thoroughly
  • Use slow motion during testing to catch issues
  • Test both headless and visual modes
  • Keep test data sets organized
Security Considerations
  • Never record with real payment information
  • Keep recordings in version control for team sharing
  • Regularly update recordings when sites change
  • Test recordings in staging environments first
  • Monitor automated purchases for anomalies
Maintenance
  • Update recordings when website layouts change
  • Test recordings monthly for reliability
  • Keep backup copies of working recordings
  • Document any site-specific quirks or requirements
